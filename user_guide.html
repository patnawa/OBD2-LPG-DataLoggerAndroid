OBD2 LPG Logger - Official User Guide

คู่มือการใช้งาน OBD2 LPG Logger

1. Introduction

Welcome to the OBD2 LPG Logger. This application is designed for automotive enthusiasts and tuners to monitor, log, and analyze live vehicle data via OBD2 interfaces (ELM327 Bluetooth/WiFi or USB Serial). It features a highly optimized LPG Mode specifically tailored for fast-polling critical tuning sensors like STFT, LTFT, RPM, and MAP.

2. Getting Started (Connection)

  1. Hardware Setup: Plug your ELM327 or OBD2 adapter into your vehicle's OBD2 port. Turn the ignition key to the ON position (engine running is optional but recommended for full data).
  2. Pairing: If using Bluetooth, pair the adapter in your Android system settings first (usually named OBDII or vLinker, PIN: 1234 or 0000).
  3. App Settings: Go to the Settings tab (Gear Icon). Select your Transport Mode (Bluetooth, WiFi, or USB). If Bluetooth is selected, pick your paired device from the dropdown.
  4. Connect: Return to the Dashboard and tap the circular Start Logging button (play icon) at the top right.

3. Dashboard & Gauges

The dashboard displays real-time data in two formats: Circular Gauges and Line Graphs.

  • Gauges: Show instant values (e.g., RPM, Vehicle Speed, Coolant Temp). Long-press a gauge to change which sensor (PID) it displays.
  • Graphs: Plot historical data over time. Useful for spotting trends in Oxygen Sensors or Fuel Trims. Long-press to change the displayed PID.

Fuel Mode Toggle: Use the "Petrol / LPG" switch on the dashboard. When switched to LPG, the app activates Fast Polling, ignoring non-essential sensors and querying only tuning-critical data to maximize the refresh rate.

4. Live ECU Fuel Correction Map

The Fuel Map tab plots average ECU fuel correction into an RPM vs. MAP/load grid. Each cell uses Cell Trim = average(STFT + LTFT) (or the one available trim). This is ECU correction, not a direct measured rich/lean AFR result.

  • Petrol Map: Collect a healthy petrol baseline first. Repair unusually high petrol correction or mechanical faults before using it as the reference.
  • LPG/CNG Map: Switch to gas and collect stable data in the same RPM/load cells and under similar conditions.
  • Data Quality: The app learns only warm (ECT ≥ 80 °C), closed-loop, stable samples with fuel-status and trim data. Gray cells have fewer than 3 stored hits; 5 or more gives better confidence, and 20 hits locks the cell. Blank means missing, filtered, or non-comparable data—not zero.
  • Δ Trim / Deviation: average LPG Trim − average Petrol Trim in the same cell. Since the inputs are percentages, Δ is measured in percentage points (pp). Positive means the ECU added more correction on gas; negative means it removed more. It suggests relative gas under-delivery or over-delivery only when conditions are comparable; it is not a measured lean/rich verdict.
  • Worked examples: Petrol +2%, LPG +8% gives Δ +6 pp (Tune Assist starts near +6%; add gas gradually). Petrol −3%, LPG −9% gives Δ −6 pp (Tune Assist starts near −6%; reduce gas gradually). Petrol +15%, LPG +15% gives Δ 0 pp, but both baselines still need investigation—zero means matched, not healthy.
  • Tune Assist: Reuses the rounded Δ number as an approximate gas-map adjustment percentage. Δ itself remains a percentage-point difference, so this is not an exact conversion or guaranteed multiplier correction. Make small changes, verify with measured Lambda/wideband, and collect a new log.
  • Diesel: Many diesel ECUs, including some Ford diesels, do not expose gasoline-style Mode 01 STFT/LTFT. A blank Trim/Δ map can be expected; use supported diesel boost, rail-pressure, MAF, EGR, and DPF data instead.

5. Diagnostic Trouble Codes (DTC)

The DTC Tab reads stored (Mode 03), pending (Mode 07), and permanent (Mode 0A) codes. Read DTCs starts a Quick/Smart scan on the protocol that already connected. We recommend leaving Smart multi-protocol scan (automatic) enabled: the app may add only standard OBD-II protocols confirmed by a previous strict Full Scan for a VIN read from this connection or explicitly entered for this live session. Turn it off to force current-protocol-only scans. A remembered VIN alone cannot authorize extra protocols or background startup expansion.

Full Scan is always manual. It checks the nine legislated OBD-II protocols (1–9), can take longer, and restores the live polling protocol when finished. It does not pretend that generic ELM USER protocols route to Ford MS-CAN or another adapter-specific physical bus. Use Full Scan once to establish a trustworthy vehicle module profile, then use Smart for routine reads.

If codes are found, repair the cause before tapping Clear DTCs. Clearing also resets emissions readiness. The engine must usually be OFF with ignition ON, and permanent codes remain until the ECU confirms the repair.

6. Data Logging & Export

Every session is automatically recorded as a `.csv` or `.jsonl` file in the selected directory (configured in Settings). You can open these files in Excel, MegaLogViewer, or custom software for deep analysis.

7. AI Agent API Server (Local REST API + SSE)

The app features a built-in Local HTTP Server designed for MCP (Model Context Protocol) Clients, AI Agents, or custom scripts to read live vehicle data and fuel-map analysis.

  • Enable: In the Settings tab, check the Enable AI Agent API Server box. Your phone's local IP address and URL will be displayed.
  • Usage: Ensure your computer/AI Agent is on the same WiFi network as the phone.
  • REST Endpoints:
    • GET /api/status - Connection status, fuel mode, VIN.
    • GET /api/data - Latest sensor data (JSON, keyed by pidKey).
    • GET /api/map - Binned fuel map with petrol/LPG/deviation/tuneAssist maps.
    • GET /api/map/summary - Map stats + tuning recommendation text.
    • GET /api/map/export - Download correction CSV (MAP kPa × RPM grid).
    • POST /api/map/import - Import a map session from JSON.
    • DELETE /api/map - Reset all map data.
    • GET /api/agent - All-in-one endpoint for AI agents: status + sensors + map summary + zone analysis (idle/cruise/acceleration/fullLoad) + hotspots (cells with |deviation| > 5% and suggested corrections) + DTCs with severity. Cached for 500ms for low-latency polling.
    • GET /api/stream - Server-Sent Events (SSE) stream — pushes per-record sensor data, map_update (current cell + trim + deviation), and map_summary (every 5 records) so AI agents get realtime updates without polling.
  • Fuel Map Data Integrity: All map data — UI, API, and SSE — reads from a single LiveMapStore with unified FLOOR-based binning. The AI agent sees exactly the same cells the user sees on screen.
  • Note: This API has zero impact on the OBD2 polling speed. It serves the latest cached data from memory.

1. บทนำ (Introduction)

ยินดีต้อนรับสู่แอป OBD2 LPG Logger แอปพลิเคชันนี้ถูกออกแบบมาเพื่อช่างจูนและผู้ใช้รถยนต์ที่ต้องการดูค่าการทำงานของเครื่องยนต์แบบเรียลไทม์ และบันทึกข้อมูล (Log) ผ่านพอร์ต OBD2 โดยมีจุดเด่นคือ โหมดแก๊ส (LPG Mode) ที่จะดึงข้อมูลด้วยความเร็วสูงพิเศษ (Fast Polling) เฉพาะเซ็นเซอร์ที่จำเป็นต่อการจูนแก๊ส (เช่น STFT, LTFT, RPM, MAP)

2. การเริ่มต้นใช้งานและการเชื่อมต่อ

  1. เสียบอุปกรณ์: เสียบตัวสแกน OBD2 (เช่น ELM327) เข้ากับพอร์ต OBD2 ของรถยนต์ จากนั้นบิดกุญแจรถไปที่ตำแหน่ง ON (หรือสตาร์ทเครื่อง)
  2. จับคู่ (Pairing): หากใช้ Bluetooth ให้เข้าไปจับคู่อุปกรณ์ในหน้าตั้งค่า Bluetooth ของมือถือก่อน (รหัสผ่านมักจะเป็น 1234 หรือ 0000)
  3. ตั้งค่าในแอป: ไปที่ หน้า Settings (ไอคอนฟันเฟือง) เลือกรูปแบบการเชื่อมต่อ (Transport Mode) เป็น Bluetooth, WiFi หรือ USB หากเลือก Bluetooth ให้เลือกชื่ออุปกรณ์ที่จับคู่ไว้
  4. เริ่มเชื่อมต่อ: กลับมาที่หน้าแรก (Dashboard) แล้วกดปุ่ม Start Logging (ไอคอน Play วงกลมมุมขวาบน)

3. หน้าปัดและกราฟ (Dashboard)

หน้า Dashboard จะแสดงข้อมูลเรียลไทม์ 2 รูปแบบคือ เกจวัด (Gauges) และ กราฟเส้น (Graphs)

  • เกจวัด: แสดงค่าปัจจุบันทันที เช่น รอบเครื่อง (RPM) ความเร็ว (Speed) สามารถ กดค้างที่เกจ เพื่อเปลี่ยนไปดูค่าเซ็นเซอร์ตัวอื่นได้
  • กราฟเส้น: แสดงแนวโน้มของข้อมูลย้อนหลัง เหมาะสำหรับดูความแกว่งของ O2 Sensor หรือ Fuel Trim กดค้างที่กราฟ เพื่อเปลี่ยนเซ็นเซอร์ได้เช่นกัน

ปุ่มสลับโหมดน้ำมัน/แก๊ส: เมื่อกดสลับเป็น LPG แอปจะเข้าสู่โหมด Fast Polling โดยตัดข้อมูลที่ไม่จำเป็นทิ้ง และดึงเฉพาะข้อมูลจูน ทำให้กราฟและเกจตอบสนองไวขึ้นมาก

4. ตารางเชื้อเพลิง (Live Fuel Map)

หน้า Fuel Map แสดงค่าเฉลี่ยการชดเชยเชื้อเพลิงของ ECU ในตารางรอบเครื่องเทียบกับ MAP/โหลด โดยใช้ Cell Trim = ค่าเฉลี่ย (STFT + LTFT) หรือใช้ค่า Trim ที่มีเพียงค่าเดียว ค่านี้ไม่ใช่ค่า AFR ที่วัดตรงหรือคำตัดสินว่าหนาหรือบาง

  • แผนที่น้ำมัน (Petrol): เก็บค่าฐานน้ำมันที่สมบูรณ์ก่อน หากค่า Trim น้ำมันสูงผิดปกติควรแก้ปัญหาทางกลก่อนนำมาเป็นฐานจูนแก๊ส
  • แผนที่แก๊ส (LPG/CNG): สลับเป็นแก๊สแล้วเก็บข้อมูลที่นิ่งในช่องรอบ/โหลดเดียวกับน้ำมันและสภาพการขับใกล้เคียงกัน
  • คุณภาพข้อมูล: แอปเรียนรู้เฉพาะตอนเครื่องอุ่น (ECT ≥ 80 °C), อยู่ใน Closed Loop, ค่าคงที่ และมีข้อมูลสถานะเชื้อเพลิงกับ Trim ช่องสีเทามีข้อมูลน้อยกว่า 3 ครั้ง, ตั้งแต่ 5 ครั้งขึ้นไปมีความมั่นใจมากขึ้น และล็อกเมื่อครบ 20 ครั้ง ช่องว่างหมายถึงข้อมูลหาย ถูกกรอง หรือเปรียบเทียบไม่ได้ ไม่ใช่ค่า 0
  • Δ Trim / Deviation: คำนวณจาก ค่า Trim เฉลี่ยแก๊ส − ค่า Trim เฉลี่ยน้ำมัน ในช่องเดียวกัน เนื่องจากค่าต้นทางเป็นเปอร์เซ็นต์ หน่วยของ Δ จึงเป็น จุดเปอร์เซ็นต์ (pp) ค่าบวกหมายถึง ECU เพิ่มการชดเชยเมื่อใช้แก๊สมากกว่า ค่าลบหมายถึงลดการชดเชยมากกว่า เป็นเพียงแนวโน้มการจ่ายแก๊สเมื่อเงื่อนไขเทียบกันได้ ไม่ใช่ผลวัดตรงว่าบางหรือหนา
  • ตัวอย่าง: น้ำมัน +2%, แก๊ส +8% ได้ Δ +6 pp (Tune Assist เริ่มใกล้ +6%; เพิ่มแก๊สทีละน้อย) น้ำมัน −3%, แก๊ส −9% ได้ Δ −6 pp (Tune Assist เริ่มใกล้ −6%; ลดแก๊สทีละน้อย) น้ำมัน +15%, แก๊ส +15% ได้ Δ 0 pp แต่ค่าฐานทั้งคู่ยังสูง—ศูนย์หมายถึงตรงกัน ไม่ได้หมายถึงสมบูรณ์
  • Tune Assist: นำตัวเลข Δ ที่ปัดเศษมาใช้เป็นเปอร์เซ็นต์ปรับตารางแก๊สโดยประมาณ ส่วน Δ ยังคงมีหน่วยเป็นจุดเปอร์เซ็นต์ จึงไม่ใช่การแปลงที่แม่นยำหรือค่าปรับ Multiplier ที่รับประกันว่าถูกต้อง ควรปรับทีละน้อย ยืนยันด้วย Lambda/wideband ที่วัดจริง และเก็บ Log ใหม่
  • เครื่องยนต์ดีเซล: ECU ดีเซลจำนวนมาก รวมถึง Ford ดีเซลบางรุ่น ไม่ส่ง STFT/LTFT แบบเบนซินผ่าน Mode 01 ตาราง Trim/Δ จึงอาจว่าง ควรใช้ข้อมูลบูสต์ แรงดันราง MAF EGR และ DPF ที่รถรองรับแทน

5. การสแกนโค้ด (DTC)

หน้า DTC อ่านโค้ดที่บันทึกไว้ (Mode 03), โค้ดรอดำเนินการ (Mode 07) และโค้ดถาวร (Mode 0A) ปุ่ม Read DTCs จะสแกนแบบ Quick/Smart บนโปรโตคอลที่เชื่อมต่อสำเร็จอยู่แล้ว แนะนำให้เปิด สแกนหลายโปรโตคอลอัตโนมัติ ไว้ แอปจะเพิ่มเฉพาะโปรโตคอล OBD-II มาตรฐานที่ Full Scan แบบเข้มงวดเคยยืนยันสำหรับ VIN ที่อ่านจากรถผ่านการเชื่อมต่อนี้ หรือ VIN ที่ผู้ใช้กรอกยืนยันสำหรับการเชื่อมต่อสดครั้งนี้ ปิดตัวเลือกนี้เมื่อต้องการบังคับให้สแกนเฉพาะโปรโตคอลปัจจุบัน VIN ที่แอปจำไว้เฉย ๆ จะไม่อนุญาตให้เพิ่มโปรโตคอลหรือขยายการสแกนอัตโนมัติตอนเริ่มระบบ

Full Scan (สแกนครบ) ต้องสั่งด้วยตนเองเสมอ โดยตรวจโปรโตคอล OBD-II มาตรฐาน 1–9 แล้วคืนค่าโปรโตคอลสำหรับดูข้อมูลสดเมื่อเสร็จ ระบบจะไม่อ้างว่า USER CAN ของ ELM ทั่วไปสามารถสลับขาไป Ford MS-CAN หรือบัสเฉพาะของอะแดปเตอร์ได้ แนะนำให้สแกนครบหนึ่งครั้งเพื่อสร้างโปรไฟล์โมดูลที่เชื่อถือได้ แล้วใช้ Smart ในการอ่านตามปกติ

หากพบโค้ด ให้ซ่อมสาเหตุก่อนกด Clear DTCs เพราะการลบโค้ดจะรีเซ็ต readiness ด้วย โดยทั่วไปต้องดับเครื่องยนต์แต่เปิดกุญแจ ON และโค้ดถาวรจะหายเมื่อ ECU ยืนยันว่าซ่อมแล้วเท่านั้น

6. การบันทึกข้อมูล (Data Logging)

ทุกครั้งที่คุณกด Start Logging แอปจะบันทึกข้อมูลทุกวินาทีลงในไฟล์ `.csv` หรือ `.jsonl` ในโฟลเดอร์ที่คุณตั้งค่าไว้ คุณสามารถนำไฟล์นี้ไปเปิดใน Excel หรือโปรแกรมวิเคราะห์ข้อมูล (เช่น MegaLogViewer) เพื่อดูข้อมูลย้อนหลังอย่างละเอียดได้

7. AI Agent API Server (เชื่อมต่อระบบ AI อัจฉริยะ + SSE Realtime)

แอปพลิเคชันนี้มีระบบ Local HTTP Server ฝังมาในตัว เพื่อให้ระบบ AI Agent, MCP Client หรือสคริปต์คอมพิวเตอร์ของคุณ สามารถดึงข้อมูลรถยนต์และตารางจูนแบบ Real-time ไปวิเคราะห์และแนะนำการปรับจูนได้

  • วิธีเปิดใช้งาน: ไปที่หน้า Settings และติ๊กถูกที่ Enable AI Agent API Server หน้าจอจะแสดง URL สำหรับดึงข้อมูลขึ้นมาให้
  • เงื่อนไข: คอมพิวเตอร์หรือระบบ AI ของคุณ จะต้องเชื่อมต่อ WiFi วงเดียวกัน กับมือถือเครื่องที่เปิดแอปนี้
  • REST API Endpoints:
    • GET /api/status - สถานะการเชื่อมต่อ, โหมดเชื้อเพลิง, VIN
    • GET /api/data - ข้อมูลเซ็นเซอร์ล่าสุด (JSON, แยกตาม pidKey)
    • GET /api/map - ตาราง Fuel Map (petrol/LPG/deviation/tuneAssist)
    • GET /api/map/summary - สถิติตาราง + คำแนะนำการจูน
    • GET /api/map/export - ดาวน์โหลด CSV ตาราง correction (MAP kPa × RPM)
    • GET /api/agent - Endpoint รวมสำหรับ AI: สถานะ + เซ็นเซอร์ + ตาราง + Zone Analysis (idle/cruise/acceleration/fullLoad) + Hotspots (cell ที่เบี่ยงเบน > 5% พร้อมคำแนะนำการแก้) + DTC พร้อมระดับความรุนแรง (แคช 500ms)
    • GET /api/stream - SSE (Server-Sent Events) ส่งข้อมูลแบบ Real-time: sensor data + map_update (cell ปัจจุบัน + trim + deviation) + map_summary (ทุก 5 records) — AI ได้ข้อมูลทันทีโดยไม่ต้อง poll
  • ความแม่นยำของตาราง: ข้อมูลตารางทั้งหมด — UI, API, และ SSE — อ่านจาก LiveMapStore แหล่งเดียว ใช้ binning แบบ FLOOR เหมือนกัน AI agent เห็นข้อมูลตรงเซลล์เดียวกับที่ผู้ใช้เห็นบนหน้าจอ
  • หมายเหตุ: การดึงข้อมูลผ่าน API นี้ จะไม่มีผลกระทบและไม่ทำให้ความเร็วในการดึงข้อมูลจากตัวสแกน OBD2 ช้าลงแต่อย่างใด
